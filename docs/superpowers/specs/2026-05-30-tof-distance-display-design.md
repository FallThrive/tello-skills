# TOF 距离显示设计

**日期**：2026-05-30
**状态**：已批准

## 背景

修改 task follow 模式下的点阵屏显示逻辑，将目标特征值（躯干高度/掩码面积）替换为 TOF 距离米数显示。

## 前次设计问题

2026-05-28 的 task_follow 修复设计文档中，矩阵显示部分存在一个遗漏问题：

**问题**：对于长文本（如 3 位数的面积或高度值），错误地使用了 `mled s` 单字符显示命令，导致 ESP32 返回 `matrix error`。

**示例**：当 `area_k = 150` 时，命令 `mled s r 150k` 会失败，因为 `mled s` 只支持单个字符显示。

**正确做法**：
- 单字符显示：`mled s r 0`、`mled s r 1`、`mled s r -`
- 多字符滚动：`mled l r 1 150k`（向左滚动，速度 1）

## 需求

1. 点阵屏只显示 TOF 距离米数的整数位（个位）
2. 无效数据（<100mm 或 8192）显示 "-"
3. 保持 2Hz 更新频率（每 0.5 秒）
4. 目标丢失后的逻辑保持现状（悬停等待）
5. 目标丢失时点阵屏显示 "?"

## 方案选择

直接修改 `controller.py` 第 1052-1068 行的显示逻辑，完全替换为目标特征值显示。

**理由**：改动最小，满足需求，无需保留目标特征值显示功能。

## 设计详情

### 修改范围

**文件**：`scripts/controller.py` 第 1052-1068 行

### 显示逻辑

```python
# 新代码
if target is None:
    # 目标丢失，显示 "?"
    self.tello.send_expansion_command("mled s r ?")
elif 100 <= tof_dist < 8192:
    # 有效距离，显示米数整数位
    meters = tof_dist // 1000  # 毫米转米，取整数
    self.tello.send_expansion_command(f"mled s r {meters}")
else:
    # 无效距离数据，显示 "-"
    self.tello.send_expansion_command("mled s r -")
```

**关键点**：
- 米数整数位（0-8）是单个字符，可以使用 `mled s` 静态显示
- 目标丢失时显示 "?"，是单个字符，同样可以使用 `mled s` 静态显示
- 避免了前次设计中长文本显示报错的问题

### 判断条件

- **目标丢失**：`target is None` → 显示 "?"
- **有效数据**：`100 ≤ tof_dist < 8192` → 显示米数
- **无效数据**：`<100`（太近不可信）或 `≥8192`（未检测到）→ 显示 "-"

### 显示内容

- 目标丢失：`?`
- 有效距离：整数米数（如 `0`、`1`、`2`、`3`）
- 无效距离：`-`

### 数据流

```
TOF 传感器 → tof_dist（mm）→ 转换判断 → 米数/无效 → 点阵屏显示
```

## 边界情况

| 场景 | 显示 | 说明 |
|------|------|------|
| 目标丢失 | `?` | target 为 None |
| TOF 获取失败 (-1) | `-` | tof_dist = -1 |
| 太近 (50mm) | `-` | <100，太近不可信 |
| 正常距离 (471mm) | `0` | 0.47 米，显示 0 |
| 正常距离 (1500mm) | `1` | 1.5 米，显示 1 |
| 正常距离 (3500mm) | `3` | 3.5 米，显示 3 |
| 未检测到 (8192) | `-` | 8192 表示未检测到 |

## 保持不变的部分

- 更新频率：2Hz（每 0.5 秒）
- 目标丢失逻辑：悬停等待（`rc 0,0,0,0`）
- 紧急停止逻辑：不变（`100 ≤ tof_dist < 5000` 触发停止）
- 目标丢失后显示：`?`（新增）

## 影响分析

**移除的功能**：
- 目标特征值显示（躯干高度/掩码面积）

**保留的功能**：
- TOF 紧急停止逻辑
- 目标跟踪和 P 控制
- 悬停等待逻辑

3. 目标丢失显示：验证目标丢失时显示 "?"
